  <name>Sequential Impulse Contact Solver</name>
  <files>src/solver/ContactSolver.ts, src/solver/SolverConstants.ts, src/solver/index.ts, tests/solver/ContactSolver.test.ts</files>
  <behavior>
    The ContactSolver operates on Manifold[] from CollisionSystem.detect(). It follows a three-phase lifecycle per physics step:

    **preStep(manifolds, dt):**
    - For each manifold, for each contact point:
      - Compute rA = contact.point - bodyA.position, rB = contact.point - bodyB.position
      - Compute effective mass for normal direction: normalMass = 1 / (invMassA + invMassB + invInertiaA * (rA x n)^2 + invInertiaB * (rB x n)^2)
      - Compute effective mass for tangent direction (same formula with tangent = Vec2(-normal.y, normal.x))
      - Compute relative velocity at contact BEFORE warm-starting: vn = relVel.dot(normal)
      - Compute bias = -(beta/dt) * max(depth - slop, 0) + restitution * max(-vn - restitutionSlop, 0)
      - Apply warm-start: apply cached normalImpulse and tangentImpulse to both bodies

    **solve():** (called N times per step, typically 8)
    - For each contact constraint:
      - Recompute relative velocity at contact point (bodies have updated velocities from prior iterations)
      - Normal impulse: lambda = normalMass * (-(vn + bias))
      - Accumulated clamping: newAccum = max(oldAccum + lambda, 0); applied = newAccum - oldAccum
      - Apply normal impulse to both bodies
      - Friction impulse: lambdaT = tangentMass * (-vt)
      - Coulomb clamping: maxFriction = friction * normalImpulse; clamp accumulated tangentImpulse to [-maxFriction, maxFriction]
      - Apply friction impulse to both bodies

    **postStep():**
    - Store accumulated impulses back into contact points (normalImpulse/tangentImpulse) for next frame's warm-start
    - (This is implicitly handled because we mutate contact.normalImpulse/tangentImpulse directly during solve)

    **SolverConstants:**
    - velocityIterations: 8
    - baumgarteFactor (beta): 0.2
    - penetrationSlop: 0.01
    - restitutionSlop: 0.5

    Test cases:
    - Two dynamic bodies colliding head-on: velocities reverse proportional to mass ratio
    - Ball on static floor with restitution=1: bounces back to original velocity magnitude
    - Ball on static floor with restitution=0: velocity zeroed after solve
    - Box on surface with high friction: tangent velocity reduced toward zero
    - Box on surface with zero friction: tangent velocity unchanged
    - Warm-start: applying cached impulse before iteration produces non-zero initial velocity correction
    - Accumulated clamping: normalImpulse never goes below zero across iterations
    - Baumgarte: penetrating contact produces positive bias pushing bodies apart
    - Static body: invMass=0 means only the dynamic body's velocity changes
  </behavior>
  <implementation>
    Create `src/solver/SolverConstants.ts` with an interface and default values:
    ```
    interface SolverConstants { velocityIterations, baumgarteFactor, penetrationSlop, restitutionSlop }
    const DEFAULT_SOLVER_CONSTANTS: SolverConstants
    ```

    Create `src/solver/ContactSolver.ts`:
    - Internal `ContactConstraint` interface storing per-contact pre-computed data: rA (Vec2), rB (Vec2), normalMass, tangentMass, bias, normal (Vec2), tangent (Vec2), references to bodyA/bodyB/contact
    - `preStep(manifolds: Manifold[], dt: number): void` -- builds ContactConstraint[] array, computes effective masses, bias, applies warm-start
    - `solve(): void` -- one iteration of normal + friction impulse solving with accumulated clamping
    - Constructor takes SolverConstants (optional, defaults to DEFAULT_SOLVER_CONSTANTS)

    CRITICAL implementation details from research:
    - Compute restitution velocity (vn) BEFORE applying warm-start impulses in preStep
    - Tangent vector computed once in preStep: `tangent = new Vec2(-normal.y, normal.x)`
    - Use inline x/y arithmetic in solve() hot loop, NOT Vec2.sub()/clone() to avoid GC pressure
    - The bias term includes BOTH Baumgarte correction AND restitution bounce
    - Accumulated clamping for normal: `newImpulse = Math.max(old + lambda, 0); applied = newImpulse - old`
    - Accumulated clamping for friction: `clamp(old + lambdaT, -maxFriction, maxFriction)`
    - When applying impulse to bodyA: subtract (impulse * invMass), subtract (r x impulse * invInertia)
    - When applying impulse to bodyB: add (impulse * invMass), add (r x impulse * invInertia)

    Create barrel export in `src/solver/index.ts`.
