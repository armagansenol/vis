    - Export `DEFAULT_WORLD_SETTINGS` constant with all defaults
    - WorldSettings should be a partial interface for the constructor (all optional) with defaults applied internally

    Create `src/engine/World.ts`:
    - Constructor takes optional Partial<WorldSettings>. Merges with DEFAULT_WORLD_SETTINGS.
    - Owns: bodies array (Body[]), CollisionSystem instance, ContactSolver instance, accumulator (number, init 0)
    - `addBody(body: Body): void` -- push to bodies array
    - `removeBody(body: Body): void` -- splice from bodies array
    - `step(frameDt: number): number` -- the fixed-timestep accumulator:
      1. Clamp frameDt: `const clamped = Math.min(frameDt, settings.maxSteps * settings.fixedDt)`
      2. Add to accumulator: `this.accumulator += clamped`
      3. While accumulator >= fixedDt, call `this.singleStep(fixedDt)`, subtract fixedDt from accumulator
      4. Return `this.accumulator / this.fixedDt` (interpolation alpha for renderer)

    `singleStep(dt: number)` -- the correct step order per research:
      1. **Integrate velocities only** (NOT positions): For each Dynamic body, apply gravity and forces to velocity:
         `vel.x += (force.x * invMass + gravity.x * gravityScale) * dt`
         `vel.y += (force.y * invMass + gravity.y * gravityScale) * dt`
         `angVel += torque * invInertia * dt`
         For Kinematic bodies: no velocity changes (user-controlled).
      2. **Detect collisions**: `const manifolds = this.collisionSystem.detect(this.bodies)`
      3. **Solve constraints**: `this.solver.preStep(manifolds, dt)` then loop velocityIterations times calling `this.solver.solve()`
      4. **Integrate positions** from corrected velocities: For each non-Static body:
         `pos.x += vel.x * dt`, `pos.y += vel.y * dt`, `angle += angVel * dt`
      5. **Clear force accumulators**: For each body: `force.set(0, 0); torque = 0`

    IMPORTANT: Do NOT call `body.integrate()` -- that method does both velocity and position in one step, but the solver needs to modify velocities between those two phases. World does the integration inline with the correct split order.

    Expose event forwarding:
    - `onBeginContact(listener)` -- delegates to CollisionSystem
    - `onEndContact(listener)` -- delegates to CollisionSystem

    Expose pair exclusion forwarding (for Phase 4 joints):
    - `addPairExclusion(idA, idB)` -- delegates to CollisionSystem
    - `removePairExclusion(idA, idB)` -- delegates to CollisionSystem
