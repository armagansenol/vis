1. **Manifold types** (`src/collision/Manifold.ts`):
   - Export interface `ContactPoint { point: Vec2; depth: number; id: number; }` where `id` is a feature ID for warm-starting persistence (edge/vertex index encoding).
   - Export interface `Manifold { bodyA: Body; bodyB: Body; normal: Vec2; contacts: ContactPoint[]; friction: number; restitution: number; isSensor: boolean; }`.
   - Normal convention: ALWAYS points from bodyA toward bodyB.
   - `friction` = `Math.sqrt(shapeA.material.friction * shapeB.material.friction)` (geometric mean).
   - `restitution` = `Math.max(shapeA.material.restitution, shapeB.material.restitution)`.
   - Export helper `function mixMaterials(a: Shape, b: Shape): { friction: number; restitution: number }`.
   - Export helper `function pairKey(idA: number, idB: number): string` returning `${min}:${max}` canonical key.

2. **Circle-circle** (in `src/collision/narrowphase.ts`):
   - Export `circleVsCircle(bodyA: Body, bodyB: Body): Manifold | null`.
   - Compute world-space centers accounting for shape offset rotated by body angle: `center = body.position + Mat2.fromAngle(body.angle).mulVec2(circle.offset)`.
   - Distance-squared check vs sum-of-radii-squared. Return null if no overlap.
   - Normal = normalized vector from centerA to centerB. Handle coincident centers with fallback normal (1, 0).
   - Depth = sumRadii - distance.
   - Contact point = centerA + normal * radiusA.
   - Single contact point with feature id = 0.
   - Set isSensor = bodyA.isSensor || bodyB.isSensor.

3. **Circle-polygon** (in `src/collision/narrowphase.ts`):
   - Export `circleVsPolygon(circleBody: Body, polyBody: Body): Manifold | null`.
   - Use Voronoi region approach:
     a. Transform circle center to polygon's local space (relative to polyBody position/angle).
     b. For each edge, find closest point on edge segment to circle center. Track minimum separation.
     c. Three cases: (i) center is inside polygon -> normal = edge normal of min penetration edge, depth = radius + signed distance; (ii) closest feature is a vertex -> vertex-circle contact; (iii) closest feature is edge interior -> edge-circle contact.
   - Transform contact point and normal back to world space.
   - Handle argument order: `detectNarrowphase` should call this regardless of which body is circle and which is polygon, swapping and flipping normal if needed.
   - Single contact point. Feature id encodes the edge/vertex index.

4. **Narrowphase dispatch** (in `src/collision/narrowphase.ts`):
   - Export `detectNarrowphase(bodyA: Body, bodyB: Body): Manifold | null`.
   - Dispatch based on `bodyA.shape.type` and `bodyB.shape.type` using ShapeType enum.
   - Circle+Circle -> `circleVsCircle`.
   - Circle+Polygon or Polygon+Circle -> `circleVsPolygon` (swap bodies if needed so circle is always first arg, then flip normal in result).
   - Polygon+Polygon -> `polygonVsPolygon` from sat.ts (implemented in Task 2).
   - For now, polygon-polygon can return null (stub) since sat.ts is Task 2.

<task type="auto">
  <name>Task 2: SAT polygon-polygon with Sutherland-Hodgman contact clipping</name>
  <files>src/collision/sat.ts, src/collision/index.ts, tests/collision/sat.test.ts</files>
  <action>
1. **SAT implementation** (`src/collision/sat.ts`):
   - Helper `getWorldVertices(body: Body, polygon: Polygon): Vec2[]`: transform each vertex by body angle (Mat2.fromAngle) and position. Account for polygon.offset.
   - Helper `getWorldNormals(body: Body, polygon: Polygon): Vec2[]`: rotate each normal by body angle.
   - Helper `projectOnAxis(vertices: Vec2[], axis: Vec2): [number, number]`: dot each vertex with axis, return [min, max].
   - Export `polygonVsPolygon(bodyA: Body, bodyB: Body): Manifold | null`:
     a. Get world vertices and normals for both polygons.
     b. For each normal of A and B: project both vertex sets onto axis, compute overlap = min(maxA, maxB) - max(minA, minB). If overlap <= 0, return null (separating axis found). Track axis with smallest positive overlap.
     c. Ensure normal points from A toward B: if `dot(normal, centerB - centerA) < 0`, flip normal.
     d. Generate contact points using Sutherland-Hodgman clipping (see below).
     e. Mix materials. Set isSensor. Return Manifold.

2. **Sutherland-Hodgman clipping for contact points:**
   - Find **reference face**: the edge on the "reference body" most aligned with the collision normal. The reference body is whichever body's normal was chosen as the MTV axis.
   - Find **incident face**: the edge on the other body most anti-aligned with the collision normal (smallest dot product with normal).
   - Clip incident face against the side planes of the reference face (two clip operations).
   - Keep clipped points that are behind the reference face plane (penetrating).
   - This produces up to 2 contact points for edge-edge contacts.
   - Assign feature IDs: encode `(referenceEdgeIndex << 8) | incidentVertexIndex` for each contact point.

3. **Update barrel** (`src/collision/index.ts`): Export from sat.ts, narrowphase.ts, Manifold.ts.

4. **Wire into dispatch**: Update `detectNarrowphase` in narrowphase.ts to call `polygonVsPolygon` instead of returning null.

5. **Tests** (`tests/collision/sat.test.ts`):
   - Two axis-aligned boxes overlapping -> correct normal (shortest axis), correct depth.
   - Two axis-aligned boxes not overlapping -> null.
   - Box on box edge-to-edge -> 2 contact points.
   - Rotated polygon vs axis-aligned box -> correct MTV for rotated case.
   - Triangle vs box -> correct collision or separation.
   - Two boxes touching at exactly one corner -> 1 contact point.
   - Pentagon vs hexagon -> correct collision detection.
   - Verify normal always points from A to B (dot product check).
   - Use `Body.resetIdCounter()` in beforeEach.
