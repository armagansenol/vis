**src/shapes/Circle.ts** (SHAP-01, BODY-09):
- Implements Shape
- Constructor: `Circle(radius: number, options?: { offset?: Vec2, density?: number, friction?: number, restitution?: number })`
- `radius: number` (readonly)
- `computeMassData(density)`:
  - area = pi * r^2
  - mass = density * area
  - inertia = 0.5 * mass * r^2 (about circle center)
  - Apply parallel axis theorem for offset: inertia += mass * offset.lengthSquared()
  - centroid = offset clone
- `computeAABB(position, angle)`:
  - Compute world-space center: position + Mat2.fromAngle(angle).mulVec2(offset)
  - min = center - (r, r), max = center + (r, r)

**src/shapes/Polygon.ts** (SHAP-02, SHAP-03):

Constructor and factories:
- `Polygon.fromVertices(vertices: Vec2[], options?)`: Clone vertices array. Validate convexity using cross-product check (see RESEARCH.md "Convexity Validation"). If not convex, throw Error with descriptive message. Enforce CCW winding: compute signed area, reverse if negative.
- `Polygon.box(width: number, height: number, options?)`: Create 4 vertices centered at origin: (-w/2,-h/2), (w/2,-h/2), (w/2,h/2), (-w/2,h/2) in CCW order. This is SHAP-02.
- `Polygon.regular(sides: number, radius: number, options?)`: Create regular polygon with N sides inscribed in circle of given radius.

Properties:
- `vertices: readonly Vec2[]` (local-space, CCW winding)
- `normals: readonly Vec2[]` (outward-facing edge normals, precomputed)
- `offset: Vec2`
- `material: Material`

Mass computation — `computeMassData(density)`:
Follow RESEARCH.md "Box2D-Style Polygon Mass Computation (Triangle Fan)" pattern exactly.
1. Use first vertex as reference point for numerical stability
2. Iterate edges, compute triangle cross products
3. Accumulate area, centroid contribution, and inertia integral
4. mass = density * area
5. centroid = accumulated / area + reference offset
6. Shift inertia to centroid
7. Apply parallel axis theorem for shape offset: inertia += mass * offset.lengthSquared()

Edge normals:
- For each edge (v[i] -> v[i+1]), compute outward normal: perpendicular of edge direction, normalized
- Precompute on construction for use in SAT collision detection later

Support function:
- `support(direction: Vec2): Vec2` — return the vertex with the maximum dot product against the direction. Used in GJK/SAT algorithms later.

`computeAABB(position, angle)`:
- Transform all vertices to world space using position + Mat2.fromAngle(angle)
- Find min/max x and y

**src/shapes/index.ts**: Barrel export Shape, ShapeType, Material, Circle, Polygon.

**tests/shapes/Polygon.test.ts**:

- Regular triangle (3 sides, r=1): verify area matches formula (3*sqrt(3)/4 * s^2 where s = side length)
- Regular hexagon: verify area close to 2.598 * r^2
- Convexity rejection: pass concave vertices (e.g., an L-shape), expect thrown error
- Winding enforcement: pass CW vertices for a square, verify they get reversed to CCW (area should be positive)
- Edge normals: for a unit square, normals should be (0,-1), (1,0), (0,1), (-1,0) (outward-facing)
- Support function: for a square, support in direction (1,1) should return the top-right vertex
- AABB: rotated box at 45 degrees should have correct expanded bounds
