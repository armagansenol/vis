**src/dynamics/Body.ts** (BODY-01, BODY-07, BODY-08, BODY-09, BODY-10):

Constructor options interface:
```typescript
interface BodyOptions {
  type?: BodyType;        // default: Dynamic
  position?: Vec2;        // default: (0,0)
  angle?: number;         // default: 0
  velocity?: Vec2;        // default: (0,0)
  angularVelocity?: number; // default: 0
  gravityScale?: number;  // default: 1
  shape: Circle | Polygon; // required
}
```

Properties:
- `type: BodyType`
- `position: Vec2`, `angle: number`
- `velocity: Vec2`, `angularVelocity: number`
- `force: Vec2` (accumulator, zeroed after each step), `torque: number` (accumulator)
- `mass: number`, `invMass: number`, `inertia: number`, `invInertia: number`
- `gravityScale: number` (BODY-10)
- `shape: Circle | Polygon`

Mass derivation on construction:
- Call `shape.computeMassData(shape.material.density)` to get mass, inertia, centroid
- For Dynamic: store mass, compute invMass = 1/mass, store inertia, compute invInertia = 1/inertia
- For Static/Kinematic: set mass=0, invMass=0, inertia=0, invInertia=0 (BODY-02, BODY-04)
- Store both mass and invMass per RESEARCH.md recommendation

Methods:

`applyForce(force: Vec2, worldPoint?: Vec2): void` (BODY-07):
- `this.force.add(force)` (always)
- If worldPoint provided: compute r = worldPoint - position, torque += r.x * force.y - r.y * force.x
- If no worldPoint: apply at center of mass (no torque)

`applyImpulse(impulse: Vec2, worldPoint?: Vec2): void` (BODY-08):
- `this.velocity.x += impulse.x * this.invMass`
- `this.velocity.y += impulse.y * this.invMass`
- If worldPoint provided: compute r = worldPoint - position, angularVelocity += (r.x * impulse.y - r.y * impulse.x) * this.invInertia

`applyForceAtCenter(force: Vec2): void` — shorthand, no torque

`setStatic()`: Change type to Static, zero out invMass/invInertia/velocity/angularVelocity
`setDynamic()`: Change type to Dynamic, recompute mass from shape
`setKinematic()`: Change type to Kinematic, zero out invMass/invInertia but keep velocity

Do NOT implement integration here — that goes in Task 2.

**Add to src/dynamics/Body.ts** (BODY-05, BODY-10):

`integrate(gravity: Vec2, dt: number): void` method:
Follow RESEARCH.md "Semi-Implicit Euler Integration" pattern exactly:
1. If type is Static, return immediately (BODY-02)
2. If type is Kinematic: only integrate position from velocity, skip force/gravity (BODY-04)
   - position.x += velocity.x * dt
   - position.y += velocity.y * dt
   - angle += angularVelocity * dt
   - return
3. If type is Dynamic (BODY-03, BODY-05):
   - velocity.x += (force.x * invMass + gravity.x * gravityScale) * dt
   - velocity.y += (force.y * invMass + gravity.y * gravityScale) * dt
   - angularVelocity += torque * invInertia * dt
   - position.x += velocity.x * dt (semi-implicit: use NEW velocity)
   - position.y += velocity.y * dt
   - angle += angularVelocity * dt
4. Clear accumulators: force.set(0, 0), torque = 0

CRITICAL: Update velocity BEFORE position. This is semi-implicit Euler. The order matters for energy conservation.
