**RevoluteConstraint.ts** â€” 2D point-to-point constraint (removes 2 translational DOF) with optional 1D angular constraint:

Constructor takes `bodyA`, `bodyB`, `worldAnchor: Vec2`, and optional `options: { enableLimit?: boolean, lowerAngle?: number, upperAngle?: number, enableMotor?: boolean, motorSpeed?: number, maxMotorTorque?: number, collideConnected?: boolean }`.
- Convert `worldAnchor` to local-space for each body: `localAnchorA = rotate(worldAnchor - bodyA.position, -bodyA.angle)`, `localAnchorB = rotate(worldAnchor - bodyB.position, -bodyB.angle)`.
- Compute `referenceAngle = bodyB.angle - bodyA.angle` at construction time.
- Defaults: `enableLimit = false`, `lowerAngle = 0`, `upperAngle = 0`, `enableMotor = false`, `motorSpeed = 0`, `maxMotorTorque = 0`, `collideConnected = false`.

Private solver fields:
- `rAx, rAy, rBx, rBy` â€” world-space lever arms (recomputed each preStep)
- `em11, em12, em22` â€” 2x2 effective mass matrix inverse (stored as 3 values since symmetric)
- `impulseX, impulseY` â€” accumulated 2D point constraint impulse
- `angularMass` â€” effective mass for angular constraint
- `angularImpulse` â€” accumulated angular impulse for limits
- `motorImpulse` â€” accumulated motor impulse
- `limitState` â€” enum: 'inactive' | 'atLower' | 'atUpper' | 'equal'

`preStep(dt)`:
1. Transform local anchors to world space (inline cos/sin math).
2. Build 2x2 K matrix (per 04-RESEARCH.md Pattern for revolute):
   - `k11 = mA + mB + iA*rAy*rAy + iB*rBy*rBy`
   - `k12 = -iA*rAx*rAy - iB*rBx*rBy`
   - `k22 = mA + mB + iA*rAx*rAx + iB*rBx*rBx`
   (where mA = invMassA, iA = invInertiaA, etc.)
3. Invert K using Cramer's rule: `det = k11*k22 - k12*k12; invDet = 1/det; em11 = k22*invDet; em12 = -k12*invDet; em22 = k11*invDet`.
4. Angular effective mass: `angularMass = 1 / (invInertiaA + invInertiaB)` (or 0 if sum is 0).
5. Angle limit state detection:
   - `currentAngle = bodyB.angle - bodyA.angle - referenceAngle`
   - If `enableLimit` and `lowerAngle == upperAngle`: state = 'equal'
   - Else if `enableLimit` and `currentAngle <= lowerAngle`: state = 'atLower'
   - Else if `enableLimit` and `currentAngle >= upperAngle`: state = 'atUpper'
   - Else: state = 'inactive', reset `angularImpulse = 0`
6. If motor not enabled, reset `motorImpulse = 0`.
7. Warm-start: apply cached `(impulseX, impulseY)` to bodies (2D), apply `(angularImpulse + motorImpulse)` to angular velocities.

`solveVelocity()`:
1. **Motor** (if enabled): Compute angular velocity error `Cdot = wB - wA - motorSpeed`. Apply `lambda = -angularMass * Cdot`. Accumulated clamping: `|motorImpulse| <= maxMotorTorque * dt` (store dt from preStep). Apply angular impulse to both bodies.

2. **Angle limit** (if active):
   - `Cdot = wB - wA`
   - `lambda = -angularMass * Cdot`
   - If state == 'equal': no clamping on angularImpulse (bilateral).
   - If state == 'atLower': `angularImpulse = max(angularImpulse + lambda, 0)` then delta.
   - If state == 'atUpper': `angularImpulse = min(angularImpulse + lambda, 0)` then delta.
   - Apply angular impulse delta to bodies.

3. **Point constraint** (always active):
   - Relative velocity at anchor: `Cdot = vB + wB x rB - vA - wA x rA` (2D vector).
   - Add Baumgarte bias: `bias = (beta/dt) * positionError` where positionError = (worldAnchorB - worldAnchorA). Compute bias in preStep, store as `biasX, biasY`.
   - `lambda = -K^-1 * (Cdot + bias)` using the stored em11, em12, em22 matrix.
   - `impulseX += lambdaX; impulseY += lambdaY` (no clamping for revolute point constraint).
   - Apply 2D impulse to both bodies (linear + angular).

All arithmetic inline x/y. No `new Vec2()` in preStep or solveVelocity.

Add public getters: `getJointAngle(): number` returns `bodyB.angle - bodyA.angle - referenceAngle`. `setMotorSpeed(speed: number)`, `setLimits(lower: number, upper: number)` for runtime tuning.

**MouseConstraint.ts** â€” 2D soft constraint from a body point to a world target position:

Constructor takes `body: Body`, `localAnchor: Vec2`, `targetWorldPoint: Vec2`, and optional `options: { hertz?: number, dampingRatio?: number, maxForce?: number }`.
- Internally: `bodyA` is a conceptual "world/ground" (not a real Body). For Constraint interface compatibility, set `bodyA = body` and `bodyB = body` (only bodyB is used in the math). Or better: store `body` as a separate field and implement the interface by accessing only the one body.
- Actually, to fit the Constraint interface cleanly: make `bodyA` a dummy reference (the same body) but only use `body` field internally. The interface requirement is just for World iteration.
- Store `localAnchor` (cloned). Store `target` as mutable Vec2 (the mouse position updates).
- Defaults: `hertz = 5.0`, `dampingRatio = 0.7`, `maxForce = 1000 * body.mass` (sensible default per pitfall 5 in research). If body.mass is 0 (static), use maxForce = 1000.
- Public method: `setTarget(worldPoint: Vec2): void` â€” updates the target position (called each frame by user).

Private solver fields:
- `rBx, rBy` â€” lever arm from body center to world anchor point
- `em11, em12, em22` â€” 2x2 effective mass matrix inverse (with gamma)
- `impulseX, impulseY` â€” accumulated 2D impulse
- `gamma, betaX, betaY` â€” soft constraint terms
- `Cx, Cy` â€” position error for bias

`preStep(dt)`:
1. Transform localAnchor to world space (inline cos/sin).
2. Compute lever arm: `rBx = worldAnchor.x - body.position.x`, `rBy = worldAnchor.y - body.position.y`.
3. Compute soft constraint terms (Catto gamma/beta for 2D):
   - `mass = body.mass` (for frequency computation, NOT invMass)
   - `omega = 2 * PI * hertz`
   - `d = 2 * mass * dampingRatio * omega`
   - `k = mass * omega * omega`
   - `gamma = 1 / (dt * (d + dt * k))`, handle zero
   - `beta = dt * k * gamma`
4. Position error: `Cx = worldAnchorX - target.x`, `Cy = worldAnchorY - target.y`.
5. Build 2x2 K matrix (only bodyB terms since bodyA is "ground"):
   - `k11 = invMass + invInertia * rBy * rBy + gamma`
   - `k12 = -invInertia * rBx * rBy`
   - `k22 = invMass + invInertia * rBx * rBx + gamma`
6. Invert K using Cramer's rule.
7. Warm-start: apply cached `(impulseX, impulseY)` to body.

`solveVelocity()`:
1. Compute velocity at anchor: `CdotX = body.velocity.x + (-body.angularVelocity * rBy)`, `CdotY = body.velocity.y + (body.angularVelocity * rBx)`.
2. Compute impulse with gamma feedback: `lambdaX = -(em11*(CdotX + beta*Cx + gamma*impulseX) + em12*(CdotY + beta*Cy + gamma*impulseY))`, `lambdaY = -(em12*(CdotX + beta*Cx + gamma*impulseX) + em22*(CdotY + beta*Cy + gamma*impulseY))`.
3. Accumulate: `oldX = impulseX; oldY = impulseY; impulseX += lambdaX; impulseY += lambdaY`.
4. Max force clamping: `mag = sqrt(impulseX^2 + impulseY^2); if (mag > maxForce * dt) { impulseX *= (maxForce * dt) / mag; impulseY *= (maxForce * dt) / mag; }`.
5. Compute applied delta: `applyX = impulseX - oldX; applyY = impulseY - oldY`.
6. Apply to body: `body.velocity.x += applyX * invMass; body.velocity.y += applyY * invMass; body.angularVelocity += (rBx * applyY - rBy * applyX) * invInertia`.

All arithmetic inline. No Vec2 allocations in hot paths. Store `dt` from preStep for use in solveVelocity max force clamping.
