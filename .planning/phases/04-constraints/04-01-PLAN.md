```typescript
interface Constraint {

}
```

**DistanceConstraint.ts** — Rigid 1D distance constraint:
- Constructor takes `bodyA`, `bodyB`, `localAnchorA: Vec2`, `localAnchorB: Vec2`, and optional `options: { length?: number, collideConnected?: boolean }`.
- If length not provided, compute from initial body positions + anchor offsets.
- Store `localAnchorA` and `localAnchorB` (clone on construction to prevent mutation).
- `collideConnected` defaults to `false`.
- Private fields for solver state: `rAx, rAy, rBx, rBy, nx, ny, mass, bias, impulse` (all init 0).
- `preStep(dt)`:
  1. Transform local anchors to world space using `cos(body.angle)` / `sin(body.angle)` inline math.
  2. Compute separation vector, distance, unit direction (nx, ny). If dist < 1e-6, set nx=0, ny=0.
  3. Compute 1D effective mass along constraint axis: `1 / (invMassA + invMassB + invInertiaA * rAxN^2 + invInertiaB * rBxN^2)`.
  4. Baumgarte bias: `(0.2 / dt) * (dist - length)`. Use 0.2 matching existing solver constant.
  5. Warm-start: apply cached `impulse` along (nx, ny) to both bodies.
- `solveVelocity()`:
  1. Compute relative velocity projected onto constraint axis (Cdot).
  2. `lambda = -mass * (Cdot + bias)`.
  3. Accumulated clamping: NO clamping for bilateral distance (maintains exact distance in both directions, not rope). Simply `impulse += lambda`.
  4. Apply lambda along (nx, ny) to both bodies using inline x/y arithmetic (no Vec2 allocations).
- Follow Box2D/Planck.js DistanceJoint pattern exactly as shown in 04-RESEARCH.md code examples.
- All arithmetic in preStep and solveVelocity must be inline x/y (no `new Vec2()` in hot paths).

**SpringConstraint.ts** — Soft 1D distance constraint with Catto gamma/beta:
- Constructor takes `bodyA`, `bodyB`, `localAnchorA: Vec2`, `localAnchorB: Vec2`, and optional `options: { length?: number, hertz?: number, dampingRatio?: number, collideConnected?: boolean }`.
- Defaults: `hertz = 1.0`, `dampingRatio = 0.5`. If length not provided, compute from initial positions.
- `collideConnected` defaults to `false`.
- Private fields: same as Distance plus `gamma`, `softMass`, `hertz`, `dampingRatio`.
- `preStep(dt)`:
  1. Same anchor transform and separation computation as DistanceConstraint.
  2. Compute 1D effective mass (same as Distance).
  3. Catto soft constraint terms (from 04-RESEARCH.md Pattern 3):
     - `omega = 2 * PI * hertz`
     - `d = 2 * mass * dampingRatio * omega` (where mass = 1/invMassSum before gamma adjustment)
     - `k = mass * omega * omega`
     - `gamma = 1 / (dt * (d + dt * k))`
     - `beta = dt * k * gamma`
     - `softMass = 1 / (invMassSum + angularTerms + gamma)`
  4. Bias = `beta * (dist - length)` (NOT Baumgarte — the spring bias is different).
  5. Warm-start with cached impulse.
- `solveVelocity()`:
  1. Compute Cdot (relative velocity along axis).
  2. `lambda = -softMass * (Cdot + bias + gamma * impulse)` (gamma feedback term is the key difference from Distance).
  3. `impulse += lambda` (no clamping for springs).
  4. Apply lambda to both bodies.
- Clamp hertz to `0.5 / dt` to prevent Nyquist instability (per pitfall 3 in research).
