**Create `examples/demos/cradle.ts`:**

Export a function that returns a DemoScene object (same interface as stacking/bouncing).

Scene factory `createCradleScene(world)`:
- Configuration: 5 balls, radius 0.5, pendulum length 5 meters, spacing = radius * 2 (balls touching at rest)
- For each ball (i = 0..4):
  - x = (i - 2) * spacing (centered around x=0)
  - Ball body: `new Circle(ballRadius, { restitution: 1.0, friction: 0 })`, position at `new Vec2(x, ballRadius)` (hanging position). BodyType.Dynamic.
  - Static anchor body: tiny `new Circle(0.05)` at `new Vec2(x, ballRadius + pendulumLength)`, BodyType.Static. (Each ball gets its own anchor for clean constraint setup.)
  - DistanceConstraint connecting anchor to ball: `new DistanceConstraint(anchor, ball, new Vec2(0, 0), new Vec2(0, 0))`. The length auto-computes from initial positions = pendulumLength.
  - Set ball.prevPosition = ball.position (copy x,y) to avoid interpolation jump.
- Release the first ball: set `balls[0].position` to `new Vec2(balls[0].position.x - 2.5, balls[0].position.y + 0.6)` (pulled to the left and slightly up to match pendulum arc). Also set `balls[0].prevPosition` to match.
- World settings for this scene: increase velocity iterations for better momentum transfer. Use `world.settings.velocityIterations = 16` if that property is exposed. Otherwise, pass higher iterations in WorldSettings constructor. Check the World constructor API.

Actually, World constructor accepts `WorldSettings`. We need to create the world with higher iterations for this scene. But the demo shell creates the World. Two approaches:
1. Have the scene factory accept the world and modify settings after creation
2. Have the scene return settings that the shell uses

Simplest: After `scene.create(world)` in the shell, the scene can mutate `world` settings. But WorldSettings may be readonly. Check if World exposes settings mutably. If not, the scene factory can just accept the world and work with default iterations. Newton's cradle will be approximately correct.

Alternative: The `DemoScene.create` could return custom `WorldSettings`, and the shell creates the World with those settings. But this changes the interface from Plan 01. Simplest: just modify `world.settings` if accessible, or accept default iterations and note in a comment that more iterations would improve accuracy.

The cleanest approach: Have the shell pass `WorldSettings` to World constructor based on scene preferences. Add an optional `worldSettings` property to DemoScene. The shell uses it: `new World({ gravity: new Vec2(0, -9.81), ...scene.worldSettings })`. Update index.html to support this. For cradle: `worldSettings: { velocityIterations: 16 }`.

Tweaks:
- "Pendulum Length" slider: min 3, max 8, step 0.5, default 5. On change, teardown and rebuild scene (simplest; pendulum length affects constraint setup). OR just note it as display-only.
- Actually, simpler tweak: "Pull Distance" — how far the first ball is pulled. min: 1, max: 4, step: 0.5, default: 2.5. This is easy to apply on reset.

**Create `examples/demos/ragdoll.ts`:**

Export a function that returns a DemoScene object.

Scene factory `createRagdollScene(world)`:
- Static ground: `Polygon.box(16, 0.5)` at `new Vec2(0, -0.25)`, BodyType.Static
- Create 2 ragdolls at different x positions: `createRagdoll(world, -2, 6)` and `createRagdoll(world, 2, 7)`
- Optionally a 3rd at `createRagdoll(world, 0, 9)` for more visual interest

`createRagdoll(world, x, y)` helper function:
- **Torso**: `Polygon.box(0.5, 1.0)` at `(x, y)`. Material: `{ density: 1, friction: 0.3, restitution: 0.1 }`
- **Head**: `new Circle(0.22)` at `(x, y + 0.72)` (top of torso + head radius)
- **Upper arms (2)**: `Polygon.box(0.5, 0.18)` at `(x +/- 0.75, y + 0.35)` (shoulder height)
- **Lower arms (2)**: `Polygon.box(0.45, 0.15)` at `(x +/- 1.35, y + 0.35)`
- **Upper legs (2)**: `Polygon.box(0.18, 0.5)` at `(x +/- 0.15, y - 0.75)` (below torso)
- **Lower legs (2)**: `Polygon.box(0.15, 0.45)` at `(x +/- 0.15, y - 1.35)`

All body parts are Dynamic with `{ density: 1, friction: 0.3, restitution: 0.05 }`.

Joints (all RevoluteConstraint with angle limits):
- **Neck**: RevoluteConstraint(torso, head, `{ x, y: y + 0.5 }`). Limits: -PI/6 to PI/6.
- **Left shoulder**: RevoluteConstraint(torso, leftUpperArm, `{ x: x - 0.25, y: y + 0.35 }`). Limits: -PI/2 to PI/3.
- **Right shoulder**: RevoluteConstraint(torso, rightUpperArm, `{ x: x + 0.25, y: y + 0.35 }`). Limits: -PI/3 to PI/2.
- **Left elbow**: RevoluteConstraint(leftUpperArm, leftLowerArm, `{ x: x - 1.0, y: y + 0.35 }`). Limits: 0 to PI * 0.75.
- **Right elbow**: RevoluteConstraint(rightUpperArm, rightLowerArm, `{ x: x + 1.0, y: y + 0.35 }`). Limits: -PI * 0.75 to 0.
- **Left hip**: RevoluteConstraint(torso, leftUpperLeg, `{ x: x - 0.15, y: y - 0.5 }`). Limits: -PI/6 to PI/3.
- **Right hip**: RevoluteConstraint(torso, rightUpperLeg, `{ x: x + 0.15, y: y - 0.5 }`). Limits: -PI/3 to PI/6.
- **Left knee**: RevoluteConstraint(leftUpperLeg, leftLowerLeg, `{ x: x - 0.15, y: y - 1.0 }`). Limits: -PI * 0.6 to 0.
- **Right knee**: RevoluteConstraint(rightUpperLeg, rightLowerLeg, `{ x: x + 0.15, y: y - 1.0 }`). Limits: 0 to PI * 0.6.

All joints: `{ enableLimit: true, lowerAngle: ..., upperAngle: ... }`.

Set prevPosition = position and prevAngle = angle for all body parts to avoid interpolation jump.

Tweaks:
- "Gravity" slider: min 0, max 20, step 0.5, default 9.81. onChange: `world.gravity.y = -value`.
