**Sidebar controls (all custom-styled for dark theme):**

- Debug toggle: calls `renderer.setDebug(on/off)`, button text reflects state
- Pause/Play: stops/starts the rAF loop. On pause, stop calling `world.step()`. On unpause, reset lastTime to avoid time spike (set `lastTime = 0` so Renderer's loop naturally handles skip-frame)
- Per-demo sliders: rendered dynamically based on `scene.tweaks` array. Style sliders with `-webkit-appearance: none`, dark track (`#2a2a2e`), accent-colored thumb (`#45b3e0`). Show label + current value.

**Scene management (TypeScript in module script):**
- Import from `'../src/index.ts'`: Vec2, Body, BodyType, Circle, Polygon, World, Renderer, MouseConstraint, DistanceConstraint, RevoluteConstraint
- Import scene factories: `createStackingScene` from `'./demos/stacking.ts'`, `createBouncingScene` from `'./demos/bouncing.ts'`
- For Newton's cradle and ragdoll, import placeholders that will be created in Plan 02. For now, create stub functions inline that just add a static ground body so tabs don't crash.
- Define `DemoScene` interface: `{ name: string, create: (world: World) => void, cleanup?: () => void, tweaks?: TweakDef[] }`
- `TweakDef`: `{ label: string, min: number, max: number, step: number, value: number, onChange: (v: number) => void }`
- `switchScene(scene)` function:
  1. If `mouseConstraint` exists, remove from world, null it
  2. Call `renderer.stop()` if renderer exists
  3. Call `currentScene.cleanup?.()` to clear timers etc.
  4. Call `Body.resetIdCounter()` to keep palette cycling predictable
  5. Create new `World({ gravity: new Vec2(0, -9.81) })`
  6. Call `scene.create(world)`
  7. Create new `Renderer(canvas, world, { scale: 50, offsetY: 50, background: '#1a1a2e', debug: debugOn })`
  8. If not paused, call `renderer.start()`
  9. Build sidebar controls from `scene.tweaks`

**Mouse interaction (pointer events on canvas):**
- `screenToWorld(sx, sy)`: inverse of Renderer's Y-up transform. `wx = (sx - offsetX) / scale`, `wy = (canvasHeight - offsetY - sy) / scale`. Use scale=50, offsetX=400, offsetY=50, canvasHeight=600.
- `bodyAtPoint(world, point)`: iterate world.getBodies(), skip Static bodies. For Circle: transform point to local body space, check distance to circle center <= radius. For Polygon: transform point to local body space, check all edge cross products (if all >= 0, point is inside CCW polygon).
- `pointerdown`: convert to world coords, find body at point. If found, compute localAnchor (worldPoint in body's local space using body.angle). Create `new MouseConstraint(body, localAnchor, worldPoint)`. Add to world. Store reference + grabbed body reference.
- `pointermove`: if active constraint, call `constraint.setTarget(screenToWorld(e.offsetX, e.offsetY))`
- `pointerup`: if active constraint, `world.removeConstraint(constraint)`, null both references

**Drag visual feedback:**
- After the Renderer draws, add a custom draw pass for drag feedback. Since Renderer.draw() is called internally by its rAF loop, use a different approach: monkey-patch or use a post-draw callback. Simplest: Instead of using `renderer.start()`, manage a custom rAF loop in the demo page that calls `world.step(dt)` and `renderer.draw(alpha)` manually, then draws the drag line/highlight on top. This gives full control.
- Actually, do this: The demo page runs its OWN rAF loop. It does NOT call `renderer.start()`. Instead:
  ```
  function loop(now) {
    if (paused) { requestAnimationFrame(loop); return; }
    if (lastTime === 0) { lastTime = now; requestAnimationFrame(loop); return; }
    let dt = (now - lastTime) / 1000;
    if (dt > 0.1) dt = 0.1;
    lastTime = now;
    const alpha = world.step(dt);
    renderer.draw(alpha);
    drawDragFeedback();
    requestAnimationFrame(loop);
  }
  ```
- `drawDragFeedback()`: If active mouseConstraint and grabbed body, get canvas 2d context, apply same Y-up transform as Renderer, draw:
  1. A white semi-transparent line (`rgba(255,255,255,0.5)`) from `constraint.getWorldAnchorA()` to `constraint.getWorldAnchorB()`, lineWidth = 1.5/scale
  2. A yellow semi-transparent outline (`rgba(255,255,100,0.6)`) around the grabbed body shape (re-draw the body outline with just stroke, thicker lineWidth = 2.5/scale)

**Important implementation notes:**
- When paused, still call `requestAnimationFrame(loop)` but skip world.step and draw. When unpausing, set `lastTime = 0` to trigger Renderer's skip-first-frame behavior.
- Tab click handlers call `switchScene(scenes[index])` and update active tab styling.
- The two stub scenes for cradle/ragdoll should have `name: "Newton's Cradle"` and `name: "Ragdoll"` with just a ground body and a comment `// TODO: Plan 02`.

**Create `examples/demos/stacking.ts`:**

Export a function that returns a `DemoScene` object (matching the interface from index.html).

Scene factory `createStackingScene(world)`:
- Static ground: `Polygon.box(16, 0.5)` at `new Vec2(0, -0.25)`, `BodyType.Static`
- Static left wall: `Polygon.box(0.5, 12)` at `new Vec2(-8, 5.75)`, `BodyType.Static`
- Static right wall: `Polygon.box(0.5, 12)` at `new Vec2(8, 5.75)`, `BodyType.Static`
- 10 boxes stacked: Each box `Polygon.box(0.9, 0.9)` with `{ friction: 0.4, restitution: 0.05 }`. Position them at `new Vec2(0, 0.25 + 0.95 * i)` for i=0..9 (each box resting just above the previous). Slightly vary x position by `(Math.random() - 0.5) * 0.02` to avoid perfect symmetry. Set prevPosition to match position.
- The scene's `tweaks` array:
  - Gravity slider: label "Gravity", min: 0, max: 30, step: 0.5, default: 9.81. onChange: update `world.gravity.y = -value` (note: gravity points DOWN, so negate)

The DemoScene object: `{ name: "Stacking Boxes", create: (w) => createStackingScene(w), tweaks: [...] }`

Note: The `tweaks` need access to the world, so structure it as: the exported function takes no args and returns a DemoScene whose `create` closure captures the tweaks' onChange references. Alternatively, have `create` return void and set up tweaks that close over the world variable. The cleanest pattern: export a function `stackingScene(): DemoScene` that builds the tweaks with a captured `world` reference that gets set when `create` is called.

**Create `examples/demos/bouncing.ts`:**

Export a function that returns a `DemoScene` object.

Scene factory `createBouncingScene(world)`:
- Static ground: `Polygon.box(16, 0.5)` at `new Vec2(0, -0.25)`, `BodyType.Static`
- Static left wall: `Polygon.box(0.5, 12)` at `new Vec2(-8, 5.75)`, `BodyType.Static`
- Static right wall: `Polygon.box(0.5, 12)` at `new Vec2(8, 5.75)`, `BodyType.Static`
- Restitution values: `[0.2, 0.4, 0.5, 0.7, 0.9]` -- 5 balls
- Staggered drop using `setTimeout`: every 600ms, create a `Circle(0.4, { restitution: r, friction: 0.1 })` body at position `new Vec2(-3.2 + i * 1.6, 8)`. Set both position and prevPosition to the same value to avoid interpolation jump.
- Store timer IDs in an array. Return a `cleanup` function that clears all timeouts.
- The scene's `tweaks` array:
  - A "Restitution Scale" slider: label "Restitution", min: 0.1, max: 1.0, step: 0.05, default: 1.0. onChange: multiply each ball's `shape.material.restitution` by scale factor. (Actually, this is tricky because balls are created over time. Simpler: label it "Drop Height" with min: 4, max: 12, step: 0.5, default: 8. This changes the y-position for subsequent drops. Store the height in a mutable variable the setTimeout callbacks read.)

**Both files import from `'../../src/index.ts'`** (Vite resolves TypeScript imports from HTML).

The DemoScene interface should be importable. Since index.html defines it inline, just match the shape: `{ name: string, create: (world: World) => void, cleanup?: () => void, tweaks?: Array<{ label: string, min: number, max: number, step: number, value: number, onChange: (v: number) => void }> }`.

**Important:**
